1. 회사의 계정과목(G/L Account) 설정
4가지 세그먼트의 경우 대기업에 적절
작은 회사는 1,2개면 커버 가능
작은 회사가 4개를 사용하게 되면 필요없게 쓰이지 않게 되고 다른 작업시 세그먼트 입력에서 불필요한 입력 발생

level - 향후 출력시 어느 level까지 출력할 것이냐를 판단 (눈여겨 볼것)

Account Add시 - 커서 밑으로 Add 됌 (반드시 커서를 넣을 곳 위에다 지정해둘것)

Heading(Title)에서는 Transaction 발동 X

External Code - 다국적 기업들이 사용시 본사에서 사용하게 되는 Account / Data exporting 시 사용
Currency - 환율
Confidential - 보안 정보시 체크하면 열람 자격 부여 가능

노란화살표 - 드릴다운 (SAP의 장점)

AR invoice의 Relationship Map을 보면 관련 정보를 도식화하여 볼 수 있음 (화면 우클릭하여 가능)

Control Account - Journal Entry에서 input 불가

bank account -> cash account 설정

계정과목 add, change 가능
고칠 때는 Edit Chart of Account 사용 => 여기서 Delete 가능

cash account에 credit balance가 있을 경우 delete 옵션 나오질 않음
(account transaction이 있는 경우 불가)

Loc. in Drawer로 위치 변경 가능

2. Journal Entry
Posting Date - 회계상의 날짜 / 회계로 costing 된 날짜
Due Date - 기간 (ex)돈 갚을 기간)
Doc. Date - 문서 input 날짜
ex) 문서 작성은 오늘이고 내보낼 날짜는 다음 달일 경우 - Doc. Date = 오늘 / Postring Date - 다음 달

tip, 스크린이 다 보이지 않을 경우 - View에서 Fit Column width 설정

자주 사용하는 것 Template 만들어두면 불러와서 사용 가능

- Accounting 하는 사람들이 다 일을 잘하는 사람들이 바로 포스팅이 되는 것, 바로 Add 됌
